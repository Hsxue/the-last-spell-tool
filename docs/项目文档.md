# The Last Spell - 地图编辑器项目文档

## 项目概述

**The Last Spell TileMap Editor** 是一个基于 React + Konva 的高性能游戏地图编辑器，专为游戏《The Last Spell》设计。该编辑器支持地形绘制、建筑放置、旗帜标记等功能，目标是在 100x100 的大型地图上保持 60fps 的流畅性能。

### 核心特性

- 🎨 **地形编辑** - 支持多种地形类型（Dirt, Stone, Crater, Empty）的绘制和擦除
- 🏗️ **建筑放置** - 141 种建筑蓝图，基于实际形状的碰撞检测
- 🚩 **旗帜系统** - 区域标记（Zone Flags）和特殊标记（Special Flags）
- 🔍 **视图控制** - 缩放（0.25x-2x）、平移、视图重置
- ⚡ **性能优化** - 视口裁剪、图层分离、网格密度过滤
- 🎯 **实时预览** - 建筑/旗帜放置预览、碰撞检测提示

#### 性能优化技术

##### 视口裁剪 (Viewport Culling)

```typescript
// src/lib/viewportCulling.ts
export function getVisibleTileRange(
  viewport: Viewport,
  mapWidth: number,
  mapHeight: number,
  tileSize: number
): { minX: number; maxX: number; minY: number; maxY: number } {
  // 计算可见范围 + 2 格安全边界
  const margin = 2;
  return {
    minX: Math.max(0, Math.floor(viewport.x / tileSize) - margin),
    maxX: Math.min(mapWidth, Math.floor((viewport.x + viewport.width) / tileSize) + margin),
    // ...
  };
}
```

### 3. 建筑蓝图系统

#### 蓝图数据结构

```typescript
interface BuildingBlueprint {
  id: string;           // 建筑 ID（如 "MagicCircle"）
  tiles: string[][];    // 2D ASCII 网格表示建筑形状
  originX: number;      // 锚点 X 坐标
  originY: number;      // 锚点 Y 坐标
}
```

#### 蓝图示例

```typescript
// L 型废墟建筑
{
  id: 'RuinsC',
  tiles: [
    ['B', 'B'],
    ['B']
  ],
  originX: 0,
  originY: 0
}

// 带空洞的神庙
{
  id: 'Temple',
  tiles: [
    ['_', 'B'],
    ['B', 'B'],
    ['_', 'B']
  ],
  originX: 0,
  originY: 1
}
```

**符号说明**：
- `'B'` - 实体方块（Building block）
- `'H'` - 可通行方块（Horizontal gate）
- `'E'` - 空方块（Empty，用于特殊建筑）
- `'_'` - 空白（完全透明，不渲染不碰撞）

#### 放置引擎

```typescript
// src/lib/placementEngine-blueprint.ts
export function canPlaceBuilding(
  buildingId: string,
  positionX: number,
  positionY: number,
  mapDimensions: MapDimensions,
  existingBuildings: PlacedBuilding[]
): PlacementResult {
  // 1. 获取蓝图
  const blueprint = getBlueprint(buildingId);
  
  // 2. 计算实际占用的 tiles
  const occupiedTiles = getOccupiedTiles(buildingId, positionX, positionY);
  
  // 3. 边界检查
  for (const [x, y] of occupiedTiles) {
    if (x < 0 || x >= mapWidth || y < 0 || y >= mapHeight) {
      return { valid: false, reason: '超出边界' };
    }
  }
  
  // 4. 碰撞检查（O(1) Set 查找）
  const occupiedSet = getAllOccupiedTiles(existingBuildings);
  for (const [x, y] of occupiedTiles) {
    if (occupiedSet.has(`${x},${y}`)) {
      return { valid: false, reason: '与现有建筑重叠' };
    }
  }
  
  return { valid: true, occupiedTiles };
}
```

## 开发规范

### 代码风格

- **TypeScript**: 严格类型检查，禁止 `as any`、`@ts-ignore`
- **React**: 函数组件 + Hooks，禁止类组件
- **状态管理**: Zustand（优先使用 selector）
- **样式**: Tailwind CSS，禁止内联样式

### 性能规范

1. **Konva 图层**：
   - 静态内容使用 `listening: false`
   - 使用 `React.memo` 包裹图层组件
   - 避免在 render 中创建新对象

2. **视口优化**：
   - 所有渲染必须使用 `getVisibleTileRange`
   - 网格线根据 zoom 过滤密度

3. **状态更新**：
   - 批量更新使用 `setTerrainBatch`
   - 避免频繁触发 `setState`

### Git 提交规范

```
feat: 添加建筑预览功能
fix: 修复网格线渲染问题
perf: 优化视口裁剪性能
docs: 更新 API 文档
```
